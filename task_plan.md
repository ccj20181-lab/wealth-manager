# Task Plan: Wealth Manager 前端精致化 + 功能可用性修复

## Goal
让项目在"面向中国用户的个人财富管理"场景下可正常使用：登录/数据流/核心流程跑通，同时整体视觉与布局达到更精致、更有辨识度的水准。

## Phases
- [x] Phase 1: 现状盘点与阻塞点定位（路由、认证、缺失页面、明显报错点）
- [x] Phase 2: 信息架构与体验设计（中国用户视角：账户/现金流/基金/目标，关键流程）
- [x] Phase 3: 实施与联调（路由与守卫、页面补齐、关键功能修复、UI 统一与精修）
- [x] Phase 4: 回归检查（TS 类型检查、关键路径手测、残留 TODO 清理）

## Key Questions
1. 本项目当前定位是"必须登录的多用户"还是"单用户免登录"？（会影响 Supabase/RLS、API 设计与页面结构）
   - **已决定**: 使用完整 Supabase Auth 邮箱认证流程
2. 中国用户关键需求优先级：账户（含支付宝/微信/信用卡）> 现金流 > 基金 > 目标/提醒，是否接受先做 MVP？
   - **已完成**: 所有核心模块已实现
3. 是否需要"体验模式/离线数据"（不配置 Supabase 也能看演示数据）？
   - **已决定**: 需要配置 Supabase 才能使用

## Decisions Made
- 从匿名登录切换到完整的邮箱认证系统（注册/登录/重置密码）
- 所有功能模块完整实现并通过类型检查
- 构建成功，可部署到生产环境
